# SwingEdge — Claude Context

## What This Project Is
Personal swing trading research and portfolio tracker for Kripal's Wealthsimple TFSA accounts (and Sushma's TFSA). Tracks holdings, generates trade plans, runs a scanner, and sends Telegram alerts. Single-user app — no multi-user auth.

## Repo
`kripals/swing-edge` on GitHub (SSH: `git@github-kripals:kripals/swing-edge.git`)

## Stack
- **Backend:** FastAPI, SQLAlchemy (async), Alembic, pydantic-settings
- **Database:** PostgreSQL — Docker locally, Supabase in production
- **Frontend:** Vue 3, Vite, Pinia, Vue Router, Axios, vite-plugin-pwa
- **Deploy:** Render (backend), Vercel (frontend), GitHub Actions (scheduler)
- **Notifications:** Telegram bot
- **Portfolio data:** SnapTrade (read-only Wealthsimple sync)
- **Market data:** Twelve Data (primary), Alpha Vantage (fallback/commodities)

## How to Run Locally

**Backend — always use Docker, never run uvicorn from terminal:**
```bash
cd swingdge-backend
docker-compose up --build -d   # start
docker-compose down            # stop
docker logs swingdge-backend-api-1 --tail 50  # logs
```

**Migrations (run once, or after schema changes):**
```bash
cd swingdge-backend
source .venv/bin/activate
alembic upgrade head
```

**Seed data (run once):**
```bash
cd swingdge-backend
source .venv/bin/activate
PYTHONPATH=. python scripts/seed.py
```

**Frontend:**
```bash
cd swingdge-frontend
npm run dev   # runs on http://localhost:5173
```

Frontend uses Vite proxy (`/api` → `http://localhost:8000`) — no CORS issues locally. Do NOT set `VITE_API_URL` in `.env` for local dev.

## Database
- **Local:** Docker PostgreSQL on `localhost:5432` (db: `swingdge`, user: `postgres`, pass: `postgres`)
- **Production:** Supabase — `DATABASE_URL` in `.env` (commented out locally)
- **Migrations:** Alembic — `alembic/versions/0001_initial_schema.py` has all 11 tables
- **Seed script:** `scripts/seed.py` — trading rules, 56 tickers, 2 accounts, 14 holdings

## Auth
Single-user JWT. Password is `APP_PASSWORD` in `.env`. Login via `POST /api/auth/login`. Token stored in `localStorage` as `swingdge_token`.

## API Keys (all in swingdge-backend/.env)
- Twelve Data — market quotes, OHLCV, indicators (800 req/day free)
- Alpha Vantage — commodities fallback (25 req/day free)
- Finnhub — fundamentals (60 req/min free)
- FMP — earnings, fundamentals (250 req/day free)
- Marketaux — news (100 req/day free)
- SnapTrade — Wealthsimple read-only portfolio sync
- Telegram — bot alerts

## Hardcoded Trading Rules (never change without thought)
- 1% risk per trade
- 2:1 minimum R/R ratio
- 5-day earnings blackout
- Max 5 active trades
- 10-day trade expiry
- $2B CAD minimum market cap
- Max 30% per sector
- Max 15% per position
- FX cost: 1.5% each way (3% round-trip) for US stocks via Wealthsimple

## TSX Tickers
Use `.TO` suffix in the app (e.g. `SU.TO`, `RY.TO`). Twelve Data requires `:TSX` format internally — the service layer handles conversion.

## Important Notes
- **Vite proxy:** Frontend calls `/api/*` which Vite proxies to `http://localhost:8000` in dev. In production, `VITE_API_URL` should point to the Render URL.
- **SnapTrade user registration:** `SNAPTRADE_USER_ID` and `SNAPTRADE_USER_SECRET_ENCRYPTED` are generated by calling `register_user()` once — not from the SnapTrade dashboard.
- **Wealthsimple linking:** Not yet connected to SnapTrade — pending backend deploy.
- **Render cold start:** ~30-60s. Frontend shows loading spinner and retries.
- **Supabase free tier:** Pauses after 1 week of inactivity — GitHub Actions daily triggers prevent this.
- **alembic.ini:** `sqlalchemy.url` is a dummy placeholder — `alembic/env.py` overrides it with `DATABASE_URL` from `.env`.
